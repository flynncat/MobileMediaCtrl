# 需求文档：UI 性能优化 & 视频缩略图修复

## 引言

当前应用存在两个严重问题：
1. **UI 严重卡顿**：拖动窗口、滚动媒体列表时非常卡，影响基本操作体验。
2. **视频缩略图不显示**：视频文件没有预览图，只有图片文件有缩略图。

经过代码分析，卡顿的根本原因是：
- 内层 `ItemsControl` 使用 `WrapPanel` 不支持 UI 虚拟化，导致所有媒体卡片（可能数百上千个）全部实例化为真实 UI 元素
- 缩略图加载没有取消机制，不可见区域的 tile 仍在加载，占用资源
- MTP 批次回调使用 `Dispatcher.Invoke` 同步调用，阻塞后台线程
- MTP 视频回退下载完整文件，阻塞串行的 MTP 通道

视频缩略图不显示的原因是：
- Shell API (`IShellItemImageFactory`) 需要在 STA 线程上调用，但 `Task.Run` 使用 MTA 线程池导致 COM 调用失败
- MTP 视频回退下载完整文件效率极低，应限制或跳过

## 需求

### 需求 1：列表虚拟化优化

**用户故事：** 作为一名用户，我希望滚动媒体列表时流畅不卡顿，以便我能快速浏览大量照片和视频。

#### 验收标准

1. WHEN 媒体列表加载完成（数百至上千个文件） THEN 系统 SHALL 仅渲染可视区域内的媒体卡片 UI 元素，不可见区域的卡片不实例化。
2. WHEN 用户滚动媒体列表 THEN 系统 SHALL 通过 UI 虚拟化回收不可见的卡片元素并复用，保持流畅的滚动帧率。
3. WHEN 媒体列表包含多个月份分组 THEN 系统 SHALL 对分组内的媒体项也实现虚拟化，而非仅对外层分组虚拟化。

### 需求 2：缩略图按需加载（懒加载）

**用户故事：** 作为一名用户，我希望只有当前可见的媒体项才加载缩略图，以便系统资源不被浪费在不可见的内容上。

#### 验收标准

1. WHEN 媒体卡片进入可视区域 THEN 系统 SHALL 开始加载该卡片的缩略图。
2. WHEN 媒体卡片离开可视区域且缩略图尚未加载完成 THEN 系统 SHALL 取消该缩略图的加载任务。
3. WHEN 用户快速滚动列表 THEN 系统 SHALL 不阻塞 UI 线程，缩略图加载在后台异步完成。
4. IF 缩略图已缓存到本地磁盘 THEN 系统 SHALL 直接从缓存加载，不重复生成。

### 需求 3：视频缩略图显示

**用户故事：** 作为一名用户，我希望视频文件也能显示预览缩略图，以便我能在列表中区分不同的视频内容。

#### 验收标准

1. WHEN 文件系统来源的视频文件需要生成缩略图 THEN 系统 SHALL 使用 Windows Shell API 在 STA 线程上提取视频帧作为缩略图。
2. WHEN MTP 来源的视频文件需要生成缩略图 THEN 系统 SHALL 优先使用 MTP 协议的 `DownloadThumbnail` 获取内置缩略图。
3. IF MTP 的 `DownloadThumbnail` 失败 THEN 系统 SHALL 跳过该视频的缩略图生成（不下载完整视频文件），显示默认占位图标。
4. WHEN 视频缩略图生成成功 THEN 系统 SHALL 将结果缓存到本地磁盘，后续直接从缓存加载。

### 需求 4：MTP 批次回调优化

**用户故事：** 作为一名用户，我希望 MTP 设备扫描过程中 UI 不被阻塞，以便我能在扫描进行时正常操作界面。

#### 验收标准

1. WHEN MTP 扫描的批次回调需要更新 UI THEN 系统 SHALL 使用 `Dispatcher.BeginInvoke`（异步）而非 `Dispatcher.Invoke`（同步），避免阻塞后台扫描线程。
2. WHEN 大量批次快速到达 THEN 系统 SHALL 合并或节流 UI 更新，避免 UI 线程被频繁的小批次更新淹没。

### 需求 5：视频文件视觉标识

**用户故事：** 作为一名用户，我希望在媒体列表中能一眼区分视频和图片，以便快速找到我需要的内容。

#### 验收标准

1. WHEN 媒体卡片对应的文件是视频 THEN 系统 SHALL 在缩略图上叠加一个播放图标（?）作为视觉标识。
2. IF 视频缩略图加载失败或不可用 THEN 系统 SHALL 显示一个默认的视频占位图标（而非空白）。

## 技术约束与边界情况

- **WPF 虚拟化限制**：`WrapPanel` 不支持虚拟化，需要替换为支持虚拟化的面板（如自定义 `VirtualizingWrapPanel` 或改用 `UniformGrid` 配合 `VirtualizingStackPanel`）。
- **Shell API 线程模型**：`IShellItemImageFactory` 是 COM 接口，必须在 STA 线程上调用。`Task.Run` 默认使用 MTA 线程池，需要创建专用 STA 线程或使用 `Dispatcher` 调度。
- **MTP 串行限制**：MTP/WPD 协议本质上是串行的，同一时刻只能有一个操作。下载完整视频文件会长时间占用 MTP 通道，阻塞其他缩略图请求。
- **内存管理**：大量 `BitmapSource` 对象需要调用 `Freeze()` 以允许跨线程访问，并避免内存泄漏。
